# VAOL v0.2.25 Release Notes

Release date: 2026-02-23

## Highlights

- Closed the remaining `v0.2.x` security parity gap for gRPC Sigstore strict verification.
- Added protobuf support for DSSE signature `rekor_entry_id` and end-to-end mapping through gRPC conversion paths.
- Added strict-online Rekor gRPC success-path regression coverage while preserving backward compatibility.

## Added

- `proto/vaol/v1/ledger.proto`:
  - `DSSESignature.rekor_entry_id` (optional, backward-compatible field addition).
- Regenerated protobuf stubs:
  - `gen/vaol/v1/ledger.pb.go`
  - `gen/vaol/v1/ledger_grpc.pb.go`
- New gRPC strict-online passing test:
  - `pkg/grpc/server_test.go` (`TestVerifyRecordStrictOnlineRekorPassesWithRekorEntryID`).

## Changed

- `pkg/grpc/convert.go` now maps:
  - `pb.DSSESignature.rekor_entry_id` <-> `signer.Signature.RekorEntryID`.
- Documentation updated for compatibility and auditor guidance:
  - `docs/api-reference.md`
  - `docs/auditor-guide.md`
- Release/version metadata bumped to `0.2.25`:
  - Python SDK (`pyproject.toml`, `vaol/__init__.py`)
  - TypeScript SDK (`package.json`, `package-lock.json`)
  - Helm chart (`deploy/helm/vaol/Chart.yaml`)

## Compatibility

- No REST endpoint schema changes.
- No gRPC method signature changes.
- No DecisionRecord schema changes.
- `rekor_entry_id` is optional and backward-compatible for gRPC clients.
- Strict-online Sigstore verification requires `rekor_entry_id` to be present.
