## Summary

`v0.2.28` closes the remaining `#22` hardening scope by adding deterministic startup-restore benchmark gating and operational alert guidance, without changing REST/gRPC schemas.

## Added

1. Startup restore benchmark gate script:
   1. `scripts/check_startup_restore_bench.sh`
   2. median `ns/op` comparison for:
      1. `BenchmarkServerStartupRestore/persisted_leaves_only`
      2. `BenchmarkServerStartupRestore/snapshot_plus_tail`
   3. deterministic output artifact: `startup-restore-bench.txt`
2. Makefile target:
   1. `make bench-startup-restore`
3. CI blocking benchmark job:
   1. `Bench (Startup Restore Gate)`
   2. artifact upload: `startup-restore-benchmark-results`
4. Startup restore observability fields in server logs:
   1. `restore_path`
   2. `restore_duration_ms`
   3. `tree_size`
   4. `checkpoint_validated`

## Changed

1. Build gate dependencies in CI now include startup-restore benchmark check.
2. Compliance/HA/DR docs now define explicit restore-performance thresholds:
   1. warning at `>120s`
   2. critical at `>300s`
   3. critical when `restore_path=record_traversal` appears more than once in 24h.
3. Release metadata updated to `0.2.28` for Python SDK, TypeScript SDK, Helm chart, and API reference examples.

## Backward Compatibility

1. No REST schema changes.
2. No protobuf/gRPC method or message changes.
3. No SDK runtime API changes.
