# VAOL v0.2.23 Release Notes

Release date: 2026-02-23

## Highlights

- Closed the remaining high-risk gRPC gap by enforcing JWT authentication and tenant isolation parity with REST.
- Added deterministic cross-tenant denial behavior for gRPC read/query/proof/export paths.
- Bound trusted authentication evidence (`auth_context`) into records on gRPC append.
- Updated architecture, threat-model, and API docs to reflect post-gRPC implementation and metadata contracts.

## Added

- gRPC auth + tenant isolation regression tests in `pkg/grpc/server_test.go` covering:
  - missing/invalid token denial (`Unauthenticated`)
  - tenant/subject mismatch denial (`PermissionDenied`)
  - cross-tenant read/proof/export denial
  - caller-tenant forcing for empty tenant list filters
  - unauthenticated `Health` behavior under required auth

## Changed

- gRPC server now validates `authorization` metadata with configured auth mode (`disabled`, `optional`, `required`) and shared JWT verifier semantics.
- Tenant-scoped gRPC RPCs now enforce claim/header tenant parity and reject missing tenant context deterministically.
- `ListRecords` and `ExportBundle` now force caller-tenant scope when request tenant filter is empty.
- Version synchronization to `0.2.23` for:
  - Python SDK
  - TypeScript SDK
  - Helm chart
  - API reference header/examples

## Fixed

- Cross-tenant evidence exposure risk on gRPC (`GetRecord`, `GetInclusionProof`, `GetProofByID`, `ExportBundle`).
- Missing auth-context parity for gRPC append path.
